# 095 — Project-wide Rust style audit

*Snapshot of every CANON Rust crate against the rules in
[`../repos/tools-documentation/rust/style.md`](../repos/tools-documentation/rust/style.md)
and the naming rule in [`../AGENTS.md`](../AGENTS.md). Three
parallel Explore agents produced first-pass punch lists; this
report folds in a verification pass that caught significant
undercounts in nota-serde-core, signal, and nexus.*

The goal is to make all code style-compliant before resuming M0
implementation work. This report enumerates the findings and
surfaces four design questions that need Li's decision before
fix-up begins.

## 4 · Design questions for Li

These four decisions block clean execution of the punch list.

### Q1 — Crate-level `from_str` / `to_string` free functions

nota-serde-core exposes free functions
[`from_str`](../repos/nota-serde-core/src/de.rs#L24),
[`from_str_nexus`](../repos/nota-serde-core/src/de.rs#L28),
[`from_str_with`](../repos/nota-serde-core/src/de.rs#L32),
`to_string`, `to_string_nexus`, `to_string_with`.

The rule says "behavior lives on types." But this signature
mirrors `serde_json::from_str` / `serde_json::to_string` — a
universally-recognised serde-ecosystem convention. Two options:

**(a)** Treat this as **exception 5** (names inherited from
well-known crates) — keep the free fns; only rename the
modules and locals. Keeps API consonant with serde_json /
serde_yaml / etc.

**(b)** Apply the rule strictly — move logic to
`Deserializer::from_str(input).into_value()` or similar; let
free fns die. Pro: maximally rule-consistent. Con: breaks
ergonomic alignment with the rest of the serde ecosystem.

The same question applies to `is_pascal_case` /
`is_lowercase_identifier` in `lexer.rs` — these are predicate
helpers, candidates for either methods on `Token` or methods on
a `Identifier` newtype.

**My read**: (a) for `from_str`/`to_string` (true serde
convention; users expect this shape), and apply the rule for
the predicate helpers (move to `Token::is_pascal_case` /
similar). But this is a judgment call worth your sign-off.

### Q2 — `…Op` type-name suffix across signal

`AssertOp` / `MutateOp` / `RetractOp` / `QueryOp` / `BatchOp`
all use the `Op` suffix. The bad→good naming table reads:

> `op` → `operation` (or specific: `assert_op`)

I read "(or specific: `assert_op`)" as: `AssertOp` is the
"specific" form and is acceptable. The compound disambiguates
what would otherwise be a bare `op`.

Alternative: rename to `AssertOperation` etc. Feels needlessly
verbose for type names that read naturally as `AssertOp` /
`MutateOp`.

**My read**: keep `…Op` suffix on type names; rename only the
bare-`op` field at [`request.rs:46`](../repos/signal/src/request.rs#L46)
and the prose-doc `op` references. Confirm.

### Q3 — Module-name renames `de.rs` / `ser.rs` → `deserializer.rs` / `serializer.rs`

This rename cascades to every `use crate::ser::…` /
`use crate::de::…` import inside the crate (~30 sites), plus
the `pub mod de` / `pub mod ser` declarations in `lib.rs`.

Mechanically straightforward but produces a wall of `use`
changes in the diff. Worth doing — file/module names are
public surface. **My read**: do it; the cost is one-shot, the
benefit is permanent.

### Q4 — `Slot` / `Revision` accessor shape

When making the field private, three constructor/accessor
shapes are reasonable:

**(a)** `pub fn new(value: u64) -> Self` + `pub fn value(&self) -> u64`.
Symmetric, clear, no trait machinery.

**(b)** `impl From<u64> for Slot` + `impl From<Slot> for u64`.
Idiomatic Rust; allows `slot.into()`.

**(c)** Both — `new` + `value` for explicit call sites,
`From`/`Into` for ergonomic conversion. Slight redundancy.

**My read**: (a) for clarity; add `From<u64>` only if a
specific call site benefits.

## 4a · Decisions on Q1–Q4 (Li, 2026-04-27)

### Q1 — Serde: build our own, staged (SUPERSEDED — see [reports/098](098-serde-replacement-decision-2026-04-27.md))

**Decision (revised after Li challenged the unilateral
keep-serde answer): build our own, staged.**

- **Stage 1 (now / through M0)**: keep serde unchanged. No
  benefit to forking mid-flight; M0 has no live serde callers
  in the daemon graph yet. **But stop adding sentinel
  wrappers.** The current six (Bind / Mutate / Negate /
  Validate / Subscribe / AtomicBatch) are the floor.
- **Stage 2 (M0 → M1 boundary)**: write hand-rolled `Decoder`
  / `Encoder` in a new crate. Reuses
  [`nota-serde-core::Lexer`](../repos/nota-serde-core/src/lexer.rs)
  only. Delete `nexus-serde` and the sentinel wrappers. Drop
  serde derive from `signal` types. `nota-serde-core` becomes
  `nota-lexer` (525 LoC tokenizer).
- **Stage 3 (rsc lands, M2+)**: rsc projects KindDecl →
  Rust struct + matching `Decoder` / `Encoder` methods.

**Why the reversal:** the earlier "keep serde with carve-outs"
answer treated the sentinel-wrapper pattern and the
QueryParser carve-out as minor cosmetic concessions. They are
not. The QueryParser carve-out is **structural** — it will
recur for every future syntactic dispatch that depends on
schema (not Rust type) identity. The sentinel wrappers are
**pure serde-fitting ceremony**. ~70% of the 1200 LoC in
nota-serde-core's `de.rs`+`ser.rs` is serde-trait scaffolding,
not format logic. The replacement is ~915 LoC vs ~1750 today,
and uniformly *our verb vocabulary* rather than half-format /
half-trait-fitting.

Full research, costing analysis, and stage breakdown in
[reports/098](098-serde-replacement-decision-2026-04-27.md).

### Q2 — `…Op` type-name suffix: rename to `…Operation`

**Decision: rename.** `AssertOp` / `MutateOp` / `RetractOp` /
`QueryOp` / `BatchOp` become `AssertOperation` /
`MutateOperation` / `RetractOperation` / `QueryOperation` /
`BatchOperation`.

Rationale per Li (2026-04-27): *"Then you'll have to change
how you think. You were taught wrong."* The
`AssertOp`-feels-fine reflex is training-data drift, not
informed judgment. The full English form is what reads as
English. The abbreviated form is ceremony to be decoded.

Backing in the strengthened naming docs:
[`mentci/AGENTS.md` §"feels too verbose" anti-pattern](../AGENTS.md)
and [`beauty-research.md`](../repos/tools-documentation/programming/beauty-research.md).

This affects:

- `signal/src/edit.rs` — `AssertOp`, `MutateOp`, `RetractOp`,
  `BatchOp`.
- `signal/src/query.rs` — `QueryOp`.
- `signal/src/request.rs:46` — the field `pub op: Box<BatchOp>`
  becomes `pub operation: Box<BatchOperation>`.
- All call sites across `signal`, `nexus`, prospective `criome`.
- Doc comments using "op" → "operation."

Estimated diff: ~80 line touches across 8 files.

### Q3 — Module rename `de.rs`/`ser.rs` → DEFERRED (files die at Stage 2)

**Decision: skip entirely.** Per the Q1 reversal, `de.rs` and
`ser.rs` will be **deleted** at the M0→M1 boundary as part of
the serde replacement (Stage 2 of [reports/098](098-serde-replacement-decision-2026-04-27.md)).
Renaming files that will be deleted is throwaway work.

The local-variable / struct-field / visitor-parameter renames
inside those files (~50 sites of `let mut de = …`,
`SeqSerializer { ser: … }`, `(self, v: V)`) are equally
throwaway. Skip those too.

**Net effect on the fix-up plan**: ~150 line touches dropped.

### Q4 — `Slot` / `Revision` accessor shape: traits + file splits

**Decision: use `From` traits, not inherent `new`/`value`
methods.** Per Li (2026-04-27): *"Use traits. Don't be afraid
of explicit code; just add a guidance to break up the code into
more files (like splitting up the traits), then we get code
that is just as maintainable while being more correct."*

For `Slot` / `Revision` / `BlsG1` / each
`nexus-serde::{Bind,Mutate,Negate,Validate,Subscribe,AtomicBatch}`:

```rust
// Field is private:
pub struct Slot(u64);

// Construction via trait:
impl From<u64> for Slot { fn from(value: u64) -> Self { Self(value) } }

// Read-out via trait:
impl From<Slot> for u64 { fn from(slot: Slot) -> u64 { slot.0 } }

// Optional borrow accessor (trait or inherent — see below):
impl AsRef<u64> for Slot { fn as_ref(&self) -> &u64 { &self.0 } }
```

Traits over inherent methods because traits compose with the
ecosystem (`.into()` works, generic functions accept `Into<u64>`,
etc.). The rule from Q4: when traits accumulate on a type, split
each trait impl into its own file (per the new
[rust/style.md §"Split traits into their own files"](../repos/tools-documentation/rust/style.md)
guidance).

For the wrapper types `Mutate<T>` / `Negate<T>` / etc., the
fix is moot — per the Q1 reversal, **all six sentinel wrappers
in `nexus-serde` are deleted at Stage 2** (they exist only to
hang serde-rename attributes; native sigil dispatch eliminates
them). Don't fix their pub-field violation now; they are
doomed code.

For Slot / Revision / BlsG1 (long-lived in `signal` and
`sema`), the From-trait pattern still applies.

Estimated diff: ~5 newtype definitions changed + new trait impls
+ ~15 call sites updated to `.into()` (the read-out pattern).
The nexus-serde 6-wrapper fix is dropped from the plan.

## 5 · Proposed order of fixes (revised after Q1 reversal)

The serde replacement (Q1 / [reports/098](098-serde-replacement-decision-2026-04-27.md))
removes substantial work from this plan. Updated order:

1. **R2 fan-out** (Slot/Revision/BlsG1 privacy + From traits)
   — touches signal, sema, lojix-store. **Skips
   nexus-serde wrappers** (deleted at Stage 2). Mechanical;
   downstream code uses `.into()` instead of `.0`. Plus the
   nota README doc fix.
2. **R12 tests-to-`tests/`** — signal/frame.rs, sema/lib.rs,
   lojix-store/layout.rs + hash.rs. Mechanical move; verify
   tests still see the right surface.
3. **nexus parse.rs free fn → method** — small, isolated.
4. **Q2 `…Op` → `…Operation` rename** — AssertOp /
   MutateOp / RetractOp / QueryOp / BatchOp across signal +
   call sites in nexus + prospective criome.

Total estimated diff: ~80 line-touches across ~8 files. No
behavior changes; tests remain green throughout.

**Dropped from this plan** (per Q1 reversal):

- Q3 module/file rename (de.rs/ser.rs files die at Stage 2).
- The local/field/parameter renames inside de.rs/ser.rs
  (same reason).
- The `nexus-serde` six-wrapper R2 fix (wrappers die at
  Stage 2).

When this resolved, restart from
[`094 §10`](094-handoff-explicit.md) (M0 step 3 — criome body)
unimpeded by serde-replacement work; the serde replacement
itself happens at the M0→M1 boundary.
